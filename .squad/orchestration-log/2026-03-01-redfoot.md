### Key Contributions

1. **15 RAG-Specific Metrics (5 Categories):**

   **Knowledge Base Health:**
   - `kb_coverage_ratio` — % of queries resulting in non-empty retrievals
   - `kb_chunk_utilization` — histogram of per-chunk retrieval frequency
   - `kb_staleness_days` — days since KB last updated

   **Retrieval Performance:**
   - `rag_retrieval_latency_ms` — p50/p95/p99 from embedding start to chunk retrieval
   - `rag_overhead_ratio` — RAG time as % of total request latency
   - `retrieved_chunk_count` — actual chunks returned vs configured topK

   **Embedding Operations:**
   - `embedding_requests_per_second` — QPS by EmbeddingAgent
   - `embedding_cost_usd` — cost attribution by agent/KB/tenant
   - `embedding_error_rate` — provider failures and rate limiting

   **Artifact Lifecycle:**
   - `artifact_weave_duration_s` — time to complete `loom weave` operation
   - `artifact_size_mb` — storage footprint of KB bundles
   - `deployment_status_distribution` — READY/PENDING/FAILED counts

   **RAG Quality Signals:**
   - `citation_click_rate` — user engagement with source citations (requires frontend)
   - `rag_relevance_score` — average pgvector similarity of top-K
   - `zero_shot_fallback_rate` — queries with no chunks retrieved

2. **Data Contracts:**
   - Extended `TraceInput` interface with RAG fields (nullable)
   - New tables: `embedding_operations`, `artifact_operations`, `chunk_retrievals` (optional)
   - Alternative: Store `retrievedChunkIds` as JSONB in traces (storage efficiency)

3. **Aggregation Strategy:**
   - Real-time dashboard queries: last 24h with partition pruning
   - Pre-aggregated hourly/daily rollups for heavy analytics
   - Materialized "hot chunks" table (top 100 most-retrieved) for performance
   - Composite indexes on `(knowledgeBaseId, created_at)`, `(embeddingAgentId, created_at)`

4. **Dashboard Placement:**
   - **Operator Dashboard:** New "RAG Analytics" section with tiles, charts, tables
   - **Tenant Portal:** KB detail analytics tab, agent RAG breakdown, new "RAG Performance" page

5. **Implementation Phasing:**
   - **Phase 1 (P0 — launch):** Traces extension, `embedding_operations` table, basic RAG tiles
   - **Phase 2 (P1 — 2 weeks):** `artifact_operations`, overhead/cost metrics, hourly rollups
   - **Phase 3 (P2 — scale):** `chunk_retrievals`, citation tracking, advanced quality signals

6. **Open Questions:**
   - Chunk retrieval granularity (separate table vs JSONB array)
   - Embedding provider pricing expansion (currently GPT-3.5/4o only)
   - Real-time vs batch metrics SLA
   - Alert thresholds (embedding error >5%, fallback >40%, overhead >30%)

## Notes

- Designed for multi-tenant observability: same metrics for both operators and tenants
- Cost tracking enables chargeback; embedding costs previously hidden
- Phasing allows MVP launch without full analytics stack
