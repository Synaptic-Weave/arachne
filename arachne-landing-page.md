# Arachne

## Docker for AI Agents

**Define it. Package it. Ship it. Run it anywhere.**

Arachne is the portable runtime for AI agents.

------------------------------------------------------------------------

## Three Things. That's It.

### 1️⃣ A Spec for Agents + Knowledge

Define your agent in one declarative file:

-   Model + provider\
-   Tools + skills\
-   Memory\
-   Knowledge bases\
-   Guardrails

Agents and knowledge are first-class artifacts --- not scattered across
code.

------------------------------------------------------------------------

### 2️⃣ Build AI Agents Like Containers

``` bash
arachne weave
```

Package your agent into a portable bundle.

-   Agent definition\
-   Model configuration\
-   Deterministic RAG artifacts\
-   Embedded knowledge references

Reproducible. Versionable. Portable.

------------------------------------------------------------------------

### 3️⃣ Publish. Run. Deploy.

``` bash
arachne push      # Publish to registry
arachne run       # Run locally
arachne deploy    # Deploy anywhere
```

-   Push to a registry\
-   Run locally for your dev cycle\
-   Deploy to your enterprise\
-   Or deploy to the hosted Arachne server

Same artifact. Every environment.

------------------------------------------------------------------------

# What the Runtime Gives You

This isn't just packaging.

## 🧠 Automatic Agentic Memory

Built-in memory with sensible defaults.

-   Conversation memory\
-   Tool state tracking\
-   Configurable persistence\
-   Swappable backends

------------------------------------------------------------------------

## 📦 Deterministic RAG Bundles

Build knowledge bases at **build time**.

-   Vectorized during `arachne weave`\
-   Versioned artifacts\
-   Immutable knowledge snapshots\
-   Referenced cleanly from your agent bundle

No re-embedding surprises in production.

## 📊 Observability Built In

No SDK patching.\
No proxy layers.\
No glue code.

Just visibility.
